When you install [Contextual Related Posts](https://webberzone.com/plugins/contextual-related-posts/), the related posts should automatically display at the end of posts and pages. However, in some cases this might not display. Below are a few options to troubleshoot this.

1. Check if this works on other posts or pages. It could be possible the plugin can’t find a related post for the current post that is being viewed.
2. Check that the setting [“Automatically add related posts to”](https://webberzone.com/support/knowledgebase/contextual-related-posts-general-settings/#automatically-add-related-posts-to) has posts and/or pages selected and you’re viewing a post or page. If you need these on a custom post type, you’ll need to select it from the same setting.
3. Clear the [plugin cache](https://webberzone.com/support/knowledgebase/contextual-related-posts-settings-tools/) and disable it by unselecting the [two cache settings](https://webberzone.com/support/knowledgebase/contextual-related-posts-general-settings/#cache-posts-only).
4. Use the [related posts shortcode](https://webberzone.com/support/knowledgebase/contextual-related-posts-shortcode/) or block to see if the related posts are displayed.
5. If the above works, check that your theme calls `the_content` tag as this is needed for the plugin to hook into and then display the related posts. If your theme doesn’t do this, then you’ll need to use the shortcode, block or template tags.
6. Install [Query Monitor and check the output generated by the plugin](https://webberzone.com/support/knowledgebase/debugging-with-query-monitor/). If you see any errors e.g. a missing fulltext index then you’ll need to regenerate it within the Tools page. Some hosts do not allow you to use fulltext indexes or this plugin.

If none of these work, please [create a ticket in the support forums](https://wordpress.org/support/plugin/contextual-related-posts/#new-topic-0) with your site URL and output of Query Monitor above for me to take a look. If you’re using the pro version, here is where you can [request support](https://webberzone.com/request-support/).
